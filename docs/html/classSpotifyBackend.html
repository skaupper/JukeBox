Detailed Description

spotify music backend class which handles api calls and starts the authentication process

all methods are capable of refreshing the token, if it gets expired

◆ setPlayback()

TResultOpt SpotifyBackend::setPlayback ( BaseTrack const &  track)
overridevirtual

Set the given track as playback (means the track gets played).

This method checks if there are any active Spotify devices (webplayer, smartphone,...), if not KeyNotFound is returned.
The next step is to check if playingDevice is set in the configuration file. If yes, and this device is currently active, it gets selected for the playback, otherwise the currently selected (Spotify Connect) device is used.

Parameters
tracktrack to play
Returns
Returns an Error on failure.
